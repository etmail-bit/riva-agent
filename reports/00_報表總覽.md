# 報表總覽：跑完 monthly-ops-refresh 之後，去哪裡看什麼（2026-07-13 建立）

這份是「導覽用」的靜態說明檔，本身不含任何當月真實數字，之後 skill 有新增功能時會更新這份，
不會有日期後綴、不會被取代——每次要確認「現在有哪些報表可以看」都可以先開這份。

---

## 1. 每次跑完，會「自動」幫你更新的東西

### 1-1. 一份彙整報表（存在 `reports/`，檔名帶當天日期）

`reports/月度總報告_<日期>.md`——由 `scripts/generate_full_report.py` 產出（2026-07-13
起，取代原本分開的 `operational_report_<日期>.md`／`staffing_cost_estimate_<日期>.md`
兩份檔案，你只要開這一份就好）。裡面依序有：

| 章節 | 內容 |
|---|---|
| 1. 月盈虧 | 兩店歷史走勢（讀 `monthly_pnl`） |
| 2. 排班建議 | 各店最新月份、預設參數算出的逐時段建議前場人力 |
| 3. 實際排班 vs 建議人力比對 | 跨月彙整平均，逐時段對照 |
| 4. 排班人力成本估算 | 保守版/積極版可省成本、逐時段落差明細（正職/兼職拆分） |
| 5. 營運報告 | 通路組合、客單價、尖峰時段、回頭客、**星期幾彙整（每日總營業額/總發票張數的中位數、最大、最小值＋發生日期）** |
| 6. 平日/假日杯數＋排班眾數 | 逐時段杯數（平日代數反推／假日真實樣本）、星期幾 x 時段排班人力眾數 |
| 7. 薪資計算（2026-07-13 新增） | 正職/兼職逐員工「公司總負擔成本」——底薪/工資＋加班費＋2026年官方勞健保/職災/勞退級距費率算出的雇主負擔，只有排班原始資料的店/月才有得算 |

**注意**：第 1~3 段（月盈虧、排班建議、實際vs建議比對）在網頁上是「可以調整參數即時試算」的
互動功能，這份報告只收錄「目前存檔預設參數」跑出來的快照，適合當存查/列印定版；
想試不同參數組合（例如改單位產能、改班別時間窗）還是要去對應網頁調整。

**第 7 段（薪資計算）精確度提醒**：兼職員工的薪資是直接拿排班原始資料的時數乘時薪算出來的，
如果那份排班資料是 AI 謄打照片、還沒人工核對過，兼職的數字只能當大概金額參考，不建議直接拿去
對帳/發薪；正職因為班別規律、逐格檢算過，精確度比較夠。職業災害保險費率目前是政府公告的
「住宿及餐飲業」大類預設值 0.19%，還沒拿公司實際的勞保局繳款單核對過。

### 1-2. 三個網頁（本機 `./scripts/run_app.sh`，即時查資料庫，不用額外動作）

| 頁面 | 權限 | 看得到什麼 |
|---|---|---|
| **月盈虧** | admin | 單店試算＋歷史走勢圖（可調參數即時試算）；「彙整」選項另外有兩店合計走勢、經營現況建議段落、通路組合/回訪次數長條圖、回頭客逐月成長趨勢折線圖 |
| **排班建議** | admin + staff | 參數可調（單位產能／開早班時數／班別時間窗／兼職時數）＋逐時段建議人力（含取整前公式，例如 `ceil(2.01)`）＋「實際排班 vs 建議人力」比對，三種範圍可切換：單月／全年彙總（排除2月）／**平日假日拆分（真實資料，2026-07-14 新增，杯量用真實星期六日單日樣本反推，涵蓋月份比全年彙總窄但都是真實數字，上下兩張圖表並列）**。沒有實際排班資料的店，「建議人力」長條照樣顯示，「實際平均人力」長條留空，不會整段隱藏 |
| **時段人力與杯數** | admin + staff | 平日/假日逐時段杯數、星期幾 x 時段實際排班人力（正職/兼職眾數）、**星期幾 x 時段：建議人力 vs 實際排班人力**（2026-07-14 新增，`scripts/estimate_staffing_by_weekday.py`：估計杯數＝發票張數×轉換比例，套建議人力公式，七天各自呈現，沒有排班原始資料的店「實際人力」留空）、星期幾 x 時段發票張數與營業額（可切換星期幾看長條/折線圖）、星期幾彙整：每日營業額/發票張數的中位數、最大值、最小值（附最大/最小發生日期） |

### 1-3. 資料庫裡的濃縮結論（不是檔案，但月盈虧「彙整」頁的建議段落會自動引用）

`store_operational_insights`／`store_staffing_insights` 兩張表：每次跑
`generate_full_report.py` 都會更新，只存聚合統計數字，不含逐筆明細。

## 2. 這份總覽「沒有」列出的東西：`reports/` 底下其他檔案

`reports/` 資料夾裡還有一些檔名格式不一樣的檔案（例如 `cups_by_daytype_2026-07-10.md`、
`invoice_weekday_holiday_2026-07-10.md`、`invoice_weekday_breakdown_2026-07-13.md` 這種）——
**這些是特定對話裡針對你當次問題臨時產出的深度分析，不是每次跑 skill 就會自動重新產生**。
裡面的數字是產出當下的資料庫快照，之後資料庫有新資料進來，這些檔案不會自動更新內容
（跟上面 1-1 那份「每次自動重跑」的整合報告不一樣）。也包含 2026-07-13 以前產生的
`operational_report_*.md`／`staffing_cost_estimate_*.md` 舊檔——這兩種檔名以後不會再新增，
舊檔只是歷史存查用，不影響最新資料的查詢。

如果你想要某個舊分析的「最新版本」，要再跟我說一次，我會用當下最新的資料重新產出，
不是自動發生的。
